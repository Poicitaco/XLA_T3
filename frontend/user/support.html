quizFrequently Asked Questions

What does the Privacy Shield do? expand_more
The Privacy Shield enables automatic clip recording when a face is detected. All recordings are encrypted using AES-256-GCM encryption. Only the administrator who set up the system can decrypt and view these clips.
Why are faces blurred in the live feed? expand_more
XLA automatically applies face anonymization to protect individuals' privacy. The anonymization mode (blur, pixelate, solid, etc.) is configured by the system administrator. Only registered family members may be excluded depending on the current privacy target setting.
How do I view encrypted clips? expand_more
Clips in the Clips section show the blurred/anonymized version directly in the browser. The fully-encrypted originals can only be decrypted by the system administrator through the Secure Archive panel using the admin password.
What camera sources are supported? expand_more
XLA supports any camera accessible to the host system — USB webcams, built-in cameras, IP cameras via RTSP, and video files. Camera selection is available from the Admin Dashboard. The system auto-detects all available sources on startup.
Can I register my face to avoid anonymization? expand_more
Yes. Ask your system administrator to register your face in the Face Registry. Once registered, you can be excluded from anonymization when the privacy target is set to "VIP / Family Excluded" mode. The system uses face embeddings for recognition.
Is my data stored on external servers? expand_more
No. XLA runs entirely on-premises. All video streams, face embeddings, and encrypted clips are stored locally on the host machine. No data is transmitted to external servers. The system operates fully offline.
The stream shows "System Offline". What should I do? expand_more
This means the privacy pipeline is not currently running. Contact your system administrator to start the pipeline from the Admin Dashboard. Once started, the live feed will appear automatically — no page reload is required.
